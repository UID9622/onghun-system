## 架构总览

龍魂系统采用**分层主权架构**，每一层都可以独立运行，也可以组合成完整系统。

```
┌─────────────────────────────────────────────────────────┐
│  L5 应用层：龍魂操作台 · CNSH 运行时 · 数字人 · 证据链插件  │
├─────────────────────────────────────────────────────────┤
│  L4 协议层：DNA 追溯 · 主权熔断 · 君子协议 · 自动收口      │
├─────────────────────────────────────────────────────────┤
│  L3 引擎层：七因子密码学 · 三才算法 · 洛书引擎 · 五行计算  │
├─────────────────────────────────────────────────────────┤
│  L2 数据层：本地优先 · 端侧加密 · 主权数据库 · 知识图谱    │
├─────────────────────────────────────────────────────────┤
│  L1 身份层：UID9622 · GPG · 设备灵魂绑定 · 行为 DNA        │
├─────────────────────────────────────────────────────────┤
│  L0 物理层：个人设备 · 私有服务器 · 国密芯片 · 离线存储    │
└─────────────────────────────────────────────────────────┘
```

## 各层职责

### L0 物理层

**原则：** 数据根留本地，物理可关。

- 个人电脑 / 手机
- 私有服务器 / NAS
- 国密 SM4/SM2 芯片支持
- 离线冷存储（可选）

### L1 身份层

**原则：** 身份不是账号，是行为 DNA。

- UID9622 主身份
- GPG 密钥对
- 设备灵魂绑定
- 七因子行为模型

### L2 数据层

**原则：** 本地优先，端侧加密，平台只传 DNA。

- SQLite / 本地知识图谱
- AES-256 / SM4 端侧加密
- 内容哈希链
- 增量同步机制

### L3 引擎层

**原则：** 用中国文化的数学骨架驱动现代算法。

- **七因子行为密码学**：行为追溯与身份验证
- **三才算法**：天、地、人三维决策模型
- **洛书引擎**：九宫格多维推演
- **五行计算器**：相生相克动态平衡

### L4 协议层

**原则：** 规则即代码，执行即约束。

- **DNA 追溯码**：每条数据自带血统证明
- **主权熔断**：越界即断，不留隐患
- **君子协议**：CC BY-NC-SA 4.0 + 非商业铁律
- **自动收口**：任务完成后自动归档留痕

### L5 应用层

**原则：** 为人民服务，不为资本表演。

- 龍魂操作台
- CNSH 中文原生脚本运行时
- 数字人 / 人格矩阵
- 证据链插件

## 核心数据流

```
用户输入
   ↓
L1 身份验证（UID9622 + GPG + 行为 DNA）
   ↓
L4 主权熔断检查（越界？→ 熔断）
   ↓
L3 引擎处理（七因子 / 三才 / 洛书 / 五行）
   ↓
L2 本地数据操作（加密存储）
   ↓
L4 DNA 追溯码生成
   ↓
L5 输出到用户 / 同步到 Notion / GitHub / CSDN
```

---

## 安全模型

龍魂系统采用**纵深防御 + 优雅降级**：

1. **身份层**：多重验证，假身份难以通过。
2. **协议层**：主权熔断，异常行为立即阻断。
3. **数据层**：端侧加密，即使设备丢失也安全。
4. **审计层**：三色审计，所有操作留痕可追溯。
5. **恢复层**：快照 + 哈希链，支持版本回退。
